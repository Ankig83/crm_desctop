# RUS шаблон: откуда берутся данные

Документ объясняет **простым языком**:
- что заполняем уже сейчас (Вариант A),
- что появится позже (расширение),
- из каких таблиц/полей это берется.

---

## 1) Как думать о программе (очень просто)

Программа делает 3 шага:

1. **Берет справочники**:
   - клиенты (`clients.xlsx`),
   - товары (`products.xlsx`),
   - акции (`promotions.xlsx`).

2. **Считает заказ**:
   - менеджер выбирает клиента,
   - выбирает товары и количество,
   - программа смотрит цену и акцию,
   - применяет скидку, если дата попала в период акции.

3. **Собирает итоговую таблицу RUS**:
   - шапка документа (клиент/доставка),
   - строки товаров,
   - итоги по сумме.

---

## 2) Вариант A (что делаем сейчас)

Вариант A = рабочий минимальный режим:

- на товар максимум 1 акция,
- скидка только в процентах,
- период акции обязателен,
- дополнительные сложные скидки из шаблона RUS пока не считаем,
- их колонки оставляем пустыми/0 или как справочные.

---

## 3) Откуда берутся данные для RUS (Вариант A)

### 3.1 Шапка документа

- **Название компании / ИНН / контакты / адрес / email**
  - источник: таблица `clients` (импорт из `clients.xlsx`)
  - выбирается тот клиент, которого менеджер выбрал перед расчетом.

- **Дата заказа**
  - источник: поле "дата расчета" в форме расчета.

- **Дата доставки**
  - пока ручной ввод менеджером или пусто (до отдельного правила).

### 3.2 Товарные строки

- **Артикул**
  - сейчас: `ID товара` из `products.xlsx`.

- **Наименование товара**
  - сейчас: `Наименование` из `products.xlsx`.

- **Цена (базовая)**
  - сейчас: `Базовая цена` из `products.xlsx`.

- **Заказ (кол-во)**
  - сейчас: ввод менеджера в форме расчета.

- **Итоговая цена строки**
  - формула Варианта A:
  - если акция есть и дата расчета внутри периода:
    `base_price * qty * (1 - discount_percent/100)`
  - иначе:
    `base_price * qty`.

### 3.3 Итоги документа

- **Итого по документу**
  - сумма всех итогов строк.

- **Регулярная сумма (без скидок)**
  - сумма `base_price * qty` по всем строкам.

- **Сумма с акцией**
  - сумма после применения скидки по Варианту A.

---

## 4) Какие колонки RUS пока НЕ считаются (Phase 2)

Это колонки типа:
- предоплата -2%,
- объем > 300 / > 500,
- акции вида 15+2 / 10+3,
- ретро-бонус,
- MOT,
- сложные доп. скидки,
- некоторые логистические производные (паллеты, масса, объем), если нет исходных параметров в каталоге.

В Варианте A они:
- либо пустые,
- либо 0,
- либо копируют базовые поля без бизнес-эффекта.

Начиная с текущей версии, дополнительные колонки «матрицы акций» при импорте
сохраняются в БД как JSON (`promotions.matrix_rules_json`) для последующего
подключения логики без повторного импорта. На расчёт цены это пока не влияет.

## 4.1) Авто-поля из шаблона RUS (что видно по пометкам заказчика)

По строке-подсказке в `RUS` (где пометки "авто из каталога", "вручную менеджером", "авто расчеты") получаем:

- **Авто из каталога (товары):**
  - артикул,
  - баркод коробки,
  - наименование,
  - ед. измерения,
  - цена за штуку (регулярная),
  - количество в коробе (шт),
  - цена короба регулярная,
  - количество на паллете (кор),
  - масса брутто,
  - объем (м3).

- **Вручную менеджером:**
  - заказ (кол-во коробов),
  - часть полей шапки (например дата доставки, если не задано правило автозаполнения).

- **Авто расчеты:**
  - цена итог (кор),
  - итоговая цена заказа,
  - агрегаты по паллетам и итогам,
  - производные колонки со скидками.

Это значит, что для полного RUS нужно расширить каталог товаров (и частично карточку клиента) этими атрибутами.

---

## 5) Что нужно добавить, чтобы это заработало "как в большом RUS"

Для расширения понадобится отдельная таблица правил скидок и параметров товара:

- `discount_rules`:
  - тип правила (предоплата, порог объема, акция N+M и т.д.),
  - условия (порог коробов, даты, флаги),
  - приоритет применения,
  - способ расчета.

- расширение `products`:
  - шт в коробе,
  - коробов на паллете,
  - масса брутто,
  - объем м3,
  - регулярная цена штуки и т.п.

- отдельный сервис расчета:
  - считает все скидки по шагам,
  - пишет в нужные колонки RUS.

---

## 6) Порядок загрузки данных в программу

Рекомендуемый порядок:

1. Загрузить `products.xlsx`.
2. Загрузить `promotions.xlsx`.
3. Загрузить `clients.xlsx`.
4. Создать расчет и выгрузить в RUS.

Почему так:
- акции ссылаются на товар по `ID товара`,
- без загруженного товара акция не привяжется.

---

## 7) Что пользователь увидит на практике

- Выбираете клиента.
- Выбираете товары и количество.
- Указываете дату расчета.
- Нажимаете "Сформировать RUS".
- Получаете Excel в формате шаблона:
  - шапка заполнена,
  - строки товара и суммы заполнены,
  - сложные колонки пока нейтральные (до Phase 2).

## 8) Важно для команды

- Этот документ описывает **Вариант A** как безопасный старт.

  но бизнес-логику включаем поэтапно, чтобы не сломать текущую работу.
