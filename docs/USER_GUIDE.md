# Руководство пользователя — CRM Desktop

**Версия программы:** 1.0  
**Последнее обновление:** апрель 2026

---

## Что это за программа?

CRM Desktop — программа для менеджеров отдела продаж. Позволяет:

- хранить базу клиентов и товаров;
- настраивать скидки и акции на каждый товар;
- быстро рассчитывать стоимость заказа с учётом всех скидок;
- формировать файл `RUS.xlsx` для автоматической загрузки заказа в 1С;
- отправлять расчёт клиенту на e-mail.

Все данные хранятся прямо на вашем компьютере — интернет не нужен (кроме отправки e-mail).

---

## Запуск программы

Откройте файл **CRM_Desktop.exe** (или ярлык на рабочем столе).

> **Где хранится база данных:**  
> `C:\Users\<Ваше имя>\AppData\Local\CRM_Desktop\crm.db`  
> База хранится отдельно от программы и не теряется при обновлении.

---

## Общие принципы работы

- **Автосохранение.** Большинство полей сохраняются автоматически — как только вы переходите к следующему полю или строке. Кнопку «Сохранить» нажимать не обязательно, но она тоже есть.
- **Поиск** есть на вкладках Клиенты и Товары — начните вводить текст, список сразу отфильтруется.
- **Импорт/Экспорт** через меню **Файл** — можно загрузить данные из Excel или выгрузить обратно.

### Вкладка «Клиенты»

Здесь хранится список всех ваших клиентов.

#### Как найти клиента

Введите часть названия компании или ИНН в строку поиска (вверху слева) — список сразу отфильтруется.

#### Кнопки панели клиентов

| Кнопка | Что делает |
|--------|-----------|
| **Новый клиент** | Создаёт новую пустую карточку клиента. После нажатия заполните поля справа — данные сохранятся автоматически. |
| **Удалить** | Удаляет выбранного клиента из базы. Программа попросит подтверждение. |
| **Сохранить** | Принудительно сохраняет все изменения в карточке. Обычно не нужна — данные сохраняются сами. |

#### Поля карточки клиента

| Поле | Пояснение |
|------|-----------|
| **ID (внешний)** | Необязательное поле. Используется при импорте/экспорте Excel для сопоставления строк. Обычно совпадает с ID клиента в вашей учётной системе. |
| **Название компании** | Полное юридическое название организации. |
| **ИНН** | 10 или 12 цифр без пробелов и тире. Программа проверит формат. |
| **Тип клиента** | Категория клиента — определяет базовую скидку при расчёте: Торговая сеть −15%, Дистрибутор −5%, Оптовик −2%, Обычный −0%. |
| **Контактное лицо** | ФИО основного контакта на стороне клиента. |
| **Электронная почта** | E-mail клиента — используется при отправке расчёта. |
| **Город / индекс** | Город и почтовый индекс клиента — попадает в RUS.xlsx. |
| **Контакты** | Телефоны и другие контакты — несколько можно ввести с новой строки. |
| **Адрес** | Юридический или фактический адрес. Если адресов несколько — каждый с новой строки. |
| **Пункты разгрузки** | Адреса, куда доставляется товар. Несколько — с новой строки. |
| **Новый клиент** ☑ | Флажок автоматически ставится при создании нового клиента. **Снимите его**, когда обработали клиента: связались, согласовали условия, отправили КП. Так вы всегда видите, кто ещё не охвачен. |

#### Блок «Грузополучатель (для RUS)»

Заполняется, если товар доставляется не самому клиенту, а другой организации (например, на склад дистрибутора).  
Данные из этого блока попадают в соответствующий раздел файла RUS.xlsx.

| Поле | Пояснение |
|------|-----------|
| Название компании | Название организации-грузополучателя |
| Контактное лицо | ФИО контакта на складе/приёмке |
| Адрес | Адрес доставки |
| Город / индекс | Город и индекс грузополучателя |
| Телефон | Телефон для связи при доставке |
| Электронная почта | E-mail грузополучателя |

### Вкладка «Товары»

Список всех товаров с ценами и характеристиками.

#### Как найти товар

Введите название или артикул в строку поиска — таблица сразу отфильтруется.

#### Кнопки

| Кнопка | Что делает |
|--------|-----------|
| **Добавить товар** | Создаёт новую пустую строку в таблице. Заполните поля прямо в ячейках. |
| **Удалить строку** | Удаляет выделенный товар и связанную с ним акцию. Программа попросит подтверждение. |

#### Колонки таблицы товаров

| Колонка | Пояснение |
|---------|-----------|
| **Артикул** | Внешний ID товара — должен совпадать с артикулом в 1С. Именно этот артикул используется в файле RUS.xlsx и в настройке бонусных акций. |
| **Баркод коробки** | Штрихкод коробки (EAN-13 или аналог). Попадает в RUS.xlsx. |
| **Наименование** | Полное название товара. |
| **Ед. изм.** | Единица измерения (обычно «кор» — коробка). |
| **Цена за штуку** | Цена одной штуки товара в рублях. Введите это значение вручную. |
| **Штук в коробке** | Сколько штук находится в одной коробке. |
| **Базовая цена (кор, авто)** | **Вычисляется автоматически** как «Цена за штуку × Штук в коробке». Поле нельзя редактировать вручную — оно пересчитывается каждый раз, когда вы меняете цену за штуку или количество штук в коробке. |
| **Кор на паллете** | Сколько коробок помещается на одной паллете. |
| **Брутто кг** | Вес одной коробки (брутто) в килограммах. |
| **Объём м³** | Объём одной коробки в кубометрах. |
| **Кор в ряде** | Количество коробок в одном ряду на паллете. |
| **Рядов в пал.** | Количество рядов на паллете. |
| **Высота пал. мм** | Высота паллеты с товаром в миллиметрах. |
| **Размер короба д\*ш\*в** | Габариты короба в формате «420\*240\*395». |

> **Совет:** логистические поля (вес, объём, размеры паллеты) нужны для выгрузки в 1С. Заполняйте их при наличии данных — это не влияет на расчёт цены.

#### Панель «Акции товара» (внизу вкладки)

Когда вы выбираете товар в таблице, внизу появляется краткая сводка по его акциям:
- Какие скидки действуют
- В какой период
- Какие бонусы предусмотрены

Это только **информационная панель** — изменить акцию можно только на вкладке **«Акции»**.

### Вкладка «Акции»

Здесь настраиваются все скидки и бонусы для каждого товара.

#### Ключевые правила

- На **один товар — только одна** запись акции.
- Скидка задаётся **только в процентах**.
- У каждой акции обязательно должны быть **дата начала** и **дата окончания** в формате **ДД.ММ.ГГГГ**.

#### Кнопки

| Кнопка | Что делает |
|--------|-----------|
| **Новая акция (выберите товар)** | Очищает форму справа и позволяет создать акцию для нового товара. Выберите товар в выпадающем списке вверху формы, заполните блоки и нажмите «Сохранить». |
| **Сохранить акцию для выбранного товара** | Записывает все заполненные блоки (скидки, бонусы, даты) в базу данных. |
| **Удалить акцию** | Удаляет акцию для выбранного товара. Данные о товаре остаются — удаляется только акция. |

#### Верхние поля формы

| Поле | Пояснение |
|------|-----------|
| **Товар** | Выберите товар из выпадающего списка, для которого настраивается акция. |
| **Базовая скидка %** | Общая скидка на этот товар в % — применяется в период действия акции. Например, 5 — значит −5% от цены. |
| **ID товаров-бонусов** | Артикулы товаров через запятую, которые могут быть выданы бонусом. Подробнее — в разделе «Акционные бонусы» ниже. |
| **Дата начала** | Первый день действия акции (включительно). Формат: **ДД.ММ.ГГГГ**. |
| **Дата окончания** | Последний день действия акции (включительно). Формат: **ДД.ММ.ГГГГ**. |

#### Блок «Скидки за предоплату»

Настройте правило: если клиент вносит предоплату не менее X% — он получает скидку Y%.

**Как добавить правило:**
1. Нажмите **«+ Порог»** — появится строка с примером (25 → 2).
2. Измените цифры: первая колонка — процент предоплаты, вторая — процент скидки.
3. Правил может быть несколько (например, 25% → −2%, 50% → −3%, 100% → −5%).
4. Нажмите **«Сохранить акцию»**.

**«− Удалить»** — удаляет выделенную строку из таблицы порогов.

> **Как работает в расчёте:** менеджер указывает процент предоплаты клиента → программа ищет самый подходящий порог → применяет скидку.

---

#### Блок «Скидки за объём заказа»

Настройте правило: если клиент заказывает не менее N коробок — он получает скидку Z%.

**Как добавить правило:**
1. Нажмите **«+ Порог»** — появится строка (300 → 6).
2. Укажите порог в коробках и размер скидки.
3. Примеры: 300 кор → −6%, 500 кор → −8%.
4. Нажмите **«Сохранить акцию»**.

> **Как работает:** суммируется общее количество коробок в заказе по данному товару → если достигнут порог, скидка применяется.

---

#### Блок «Продуктовая скидка»

Применяется к конкретному товару независимо от объёма и предоплаты.

| Поле | Пояснение |
|------|-----------|
| **Скидка %** | Постоянная скидка на этот товар в %. Применяется всегда, если значение больше 0. |
| **Срок годности %** | Дополнительная скидка в % для товаров с коротким/истекающим сроком годности. Проставляется менеджером ситуационно. Значение попадает в колонку «Доп. скидка» файла RUS.xlsx. |

---

#### Блок «Акционные бонусы (купи → получи бесплатно)»

Настройте правила: при покупке N коробок клиент получает что-то бесплатно.

**Как добавить правило:**
1. Нажмите **«+ Правило»** — появится строка с порогом 50.
2. Заполните нужные ячейки (см. таблицу ниже).
3. Нажмите **«Сохранить акцию»**.

**«− Удалить»** — удаляет выделенное правило из таблицы.

##### Колонки таблицы бонусов

| Колонка | Что означает | Пример |
|---------|-------------|--------|
| **Порог (кор)** | Минимальное количество коробок, которое нужно купить, чтобы сработал бонус | `10` |
| **Тот же +кор** | Сколько коробок **этого же** товара добавляется бесплатно автоматически — без вопросов | `2` |
| **Конкретный (арт.)** | Артикул **другого товара**, который добавляется бесплатно автоматически. Артикул — это значение из колонки «Артикул» в таблице товаров | `batonic` |
| **Кол-во** | Сколько коробок этого конкретного другого товара дарим | `1` |
| **На выбор (арт., запятая)** | Список артикулов через запятую — менеджер в момент расчёта спросит у клиента, какой товар выбрать, и добавит его бесплатно | `P06,P07,P08` |

> **Важно:** всё, что заполнено в одной строке, срабатывает одновременно. Если заполнены и «Тот же +кор» и «На выбор» — клиент получит оба бонуса. Если нужен только один — заполните только одну ячейку.

---

##### Как работает логика порогов

> **Главное правило: из всех подходящих порогов срабатывает только ОДИН — самый высокий.**

Пример: у товара два правила — порог 10 и порог 15.

- Клиент берёт **12 коробок** → 10 ≤ 12 ✓, 15 ≤ 12 ✗ → срабатывает **правило 10**
- Клиент берёт **16 коробок** → 10 ≤ 16 ✓, 15 ≤ 16 ✓ → срабатывает **правило 15** (оно лучше)
- Клиент берёт **9 коробок** → ни один порог не достигнут → **бонусов нет**

Правила **не суммируются**. Всегда работает только одно — лучшее из достигнутых.

##### Примеры настройки бонусов

**Вариант 1 — «10+1»: купи 10, получи 1 того же товара**

| Порог | Тот же +кор | Конкретный | Кол-во | На выбор |
|-------|-------------|-----------|--------|----------|
| 10    | 1           |           |        |          |

→ Купил 10+ кор → +1 коробка того же товара добавляется автоматически.

---

**Вариант 2 — «10+1 на выбор»: купи 10, выбери 1 из списка**

| Порог | Тот же +кор | Конкретный | Кол-во | На выбор |
|-------|-------------|-----------|--------|----------|
| 10    | 0           |           |        | P01,P02,P03 |

→ Купил 10+ кор → в расчёте появится диалог «Какой товар выбирает клиент?» → менеджер выбирает → товар добавляется бесплатно.

---

**Вариант 3 — «50+1 конкретного»: купи 50, получи 1 коробку другого товара**

| Порог | Тот же +кор | Конкретный | Кол-во | На выбор |
|-------|-------------|-----------|--------|----------|
| 50    | 0           | batonic   | 1      |          |

→ Купил 50+ кор → автоматически добавляется 1 коробка «batonic» бесплатно.

---

**Вариант 4 — «100+2 конкретных + 1 на выбор»: сложная акция**

| Порог | Тот же +кор | Конкретный | Кол-во | На выбор |
|-------|-------------|-----------|--------|----------|
| 100   | 0           | batonic   | 2      | P06,P07,P08 |

→ Купил 100+ кор → автоматически +2 коробки batonic + диалог выбора 1 товара из P06, P07, P08.  
В расчёте будет **две** бонусные строки — это нормально.

---

**Пример таблицы с двумя уровнями порогов:**

| Порог | Тот же +кор |
|-------|-------------|
| 10    | 1           |
| 15    | 2           |

→ Купил 12 кор → +1 коробка  
→ Купил 16 кор → +2 коробки  
→ Купил 9 кор → бонусов нет

---

##### Частая ошибка: две бонусные строки вместо одной

Если в расчёте появились **две строки бонуса** там, где вы ожидали одну — проверьте: в строке таблицы бонусов заполнены **и «Тот же +кор»** (не 0) **и «На выбор»** (не пусто). Это даёт два бонуса одновременно.

| Порог | **Тот же +кор** | На выбор | Результат |
|-------|-----------------|----------|-----------|
| 10    | **1**           | P01,P02  | ❌ 2 бонусные строки |
| 10    | **0**           | P01,P02  | ✅ 1 строка — только на выбор |
| 10    | **1**           |          | ✅ 1 строка — только того же |

---

#### Период действия бонусной акции

Внизу блока «Акционные бонусы» можно задать отдельный период для бонусов:

| Опция | Что делает |
|-------|-----------|
| **Дата начала** + поле даты | Бонус активен с указанной даты |
| **Без даты начала (сразу активно)** ☑ | Бонус активен с самого начала, без ограничения по старту |
| **Дата окончания** + поле даты | Бонус активен до указанной даты |
| **Бессрочно (без даты окончания)** ☑ | Бонус действует бессрочно |

### Вкладка «Расчёт» — главная рабочая вкладка

Здесь формируется заказ и выгружаются документы.

#### Как составить заказ — пошаговая инструкция

**Шаг 1. Выберите клиента**  
В выпадающем списке «Клиент» выберите нужную компанию.  
Программа автоматически подставит тип клиента и его скидку.

**Шаг 2. Укажите даты**  
- **Дата расчёта** — дата формирования заказа. По умолчанию ставится сегодняшняя дата. По этой дате программа проверяет, попадает ли заказ в период действия акций.
- **Дата доставки** — когда товар должен быть доставлен клиенту. Обязательное поле для RUS.xlsx (нужно для 1С).

**Шаг 3. Проверьте номер заказа**  
Номер подставляется автоматически (следующий по порядку). При необходимости измените вручную.

**Шаг 4. Укажите предоплату (%)**  
Введите процент предоплаты, которую готов внести клиент (0–100%).  
Программа автоматически подберёт скидку за предоплату, если она настроена в акции.

**Шаг 5. Добавьте товары**  
Нажмите **«+ Добавить товар»**, выберите нужный товар из списка, укажите количество коробок.  
Программа автоматически:
- применит скидку за тип клиента;
- проверит, действует ли акция на эту дату — и применит базовую скидку акции;
- подберёт скидку за предоплату;
- подберёт скидку за объём;
- добавит бонусные строки (если достигнут порог).

**Шаг 6. Проверьте итог**  
В таблице расчёта видны все позиции, цены, скидки и итоговые суммы.

**Шаг 7. Сформируйте документ / отправьте**  
Используйте нужную кнопку экспорта (см. ниже).

#### Как работают бонусы в расчёте

Когда вы указываете количество коробок для товара с настроенными бонусами:

- **Тот же товар бесплатно** — строка с бонусным товаром добавляется автоматически с ценой 0. Ничего делать не нужно.
- **Конкретный другой товар** — тоже добавляется автоматически с ценой 0.
- **На выбор** — появится диалоговое окно: «Какой товар выбирает клиент?». Выберите нужный — он добавится с ценой 0.

> Бонусные строки выделены в таблице и не редактируются. Если изменить количество основного товара — бонусы пересчитываются автоматически.

#### Несколько порогов подряд (кратность подарка)

Если правило звучит как «каждые **10** коробок — **+1** коробка в подарок», то при **20** коробках программа начислит **2** бесплатные, при **30** — **3** и т.д.  
Учитываются только **полные** пороги: например, **19** коробок дадут такой же подарок, как **10** (один «полный» порог).

#### Цена за штуку = 0

Сумма за коробку в расчёте считается **точно так же**, как во вкладке «Товары»: цена за штуку × количество штук в коробке. Если цена за штуку **ноль**, то и итог по этой позиции будет **ноль** (программа не подставит другую цену втайне).

#### Адрес для «Отправить на e-mail»

После **успешной** отправки письма программа **сохраняет введённый адрес**. В следующий раз он уже появится в окне — можно отправить снова тому же клиенту или заменить адрес.

## Часто задаваемые вопросы

**Q: Данные потеряются при обновлении программы?**  
A: Нет. База данных хранится отдельно от программы в папке `AppData\Local\CRM_Desktop`.

**Q: Можно работать без интернета?**  
A: Да, всё работает полностью офлайн, кроме отправки e-mail.

**Q: Почему базовая цена коробки не редактируется?**  
A: Она рассчитывается автоматически: Цена за штуку × Штук в коробке. Чтобы изменить цену коробки — измените цену за штуку или количество штук.

**Q: Что означает галочка «Новый клиент»?**  
A: Она ставится автоматически при создании клиента. Снимите её, когда обработали нового клиента: связались, согласовали условия, отправили КП. Так вы видите, кто ещё ждёт вашего внимания.

**Q: Артикул в бонусах — это тот же артикул, что в таблице товаров?**  
A: Да, это значение из колонки «Артикул» на вкладке «Товары». Именно его нужно вводить в полях «Конкретный (арт.)» и «На выбор» при настройке бонусов. Убедитесь, что артикулы заполнены у всех товаров.

**Q: Почему в расчёте скидка не применяется?**  
A: Проверьте: 1) Дата расчёта входит в период действия акции. 2) У товара указан артикул. 3) Акция сохранена (нажата кнопка «Сохранить акцию»).

**Q: Файл RUS.xlsx не загружается в 1С?**  
A: Убедитесь, что в расчёте: указана дата доставки; у клиента заполнен ИНН; у всех товаров указан артикул, совпадающий с 1С.

**Q: Что делать если программа не запускается?**  
A: Убедитесь, что установлен Visual C++ Redistributable. Если не помогло — обратитесь к разработчику.
